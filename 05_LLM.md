## 5.1 什么是LLM，规模定律 / What is LLM & Scaling Laws

### 🧠 一句话理解
> LLM是在极大规模文本数据上训练的超大Transformer模型，其核心能力来自"规模"——参数越多、数据越多、算力越多，能力越强。

### LLM和普通语言模型的区别

| 对比 | 普通语言模型（如BERT） | 大语言模型（LLM） |
|------|---------------------|----------------|
| 参数量 | 亿级（1.1亿 BERT-base）| 百亿~万亿级（GPT-4约1.8万亿）|
| 训练数据 | GB级 | TB~PB级（整个互联网）|
| 能力 | 专用（分类/NER）| 通用（对话/推理/编程/创作）|
| 涌现能力 | 无 | 有（达到一定规模才突然出现）|

### 规模定律 Scaling Laws（Chinchilla Law）

#### 🧠 一句话理解
> 模型性能与"参数量 × 数据量 × 算力"呈可预测的幂律关系，三者要均衡搭配。

```
Kaplan et al.(2020) OpenAI 发现的规律：
  模型损失 ∝ N^(-0.076) × D^(-0.095) × C^(-0.050)
  N=参数量, D=数据量, C=算力

Hoffmann et al.(2022) DeepMind Chinchilla规律：
  最优配置：训练token数 ≈ 20 × 参数量
  
  例：70亿参数模型 → 最优训练数据 = 140B tokens（1400亿词）
  
  启示：之前很多大模型"训练不足"，不是越大越好，
         而是"越大越需要配套的数据量"
```

### 涌现能力 Emergent Abilities

#### 🧠 一句话理解
> 某些能力不会随模型变大线性增长，而是在达到某个临界规模后"突然"出现。

```
涌现的例子：
  小模型：完全不会做三位数加法
  中等模型：会做简单加法
  大模型（GPT-3级别）：突然会做算术、逻辑推理、写代码...

这就像水温升到100°C才沸腾——不是渐变，是突变。
```

#### ❓ 面试常问
**Q：什么是LLM的涌现能力？**
> 💡 "涌现能力是指当模型规模超过某个阈值时，新能力会突然出现，而不是随规模线性增长。典型例子是Chain-of-Thought推理——小模型完全不会，GPT-3达到一定规模后突然能够进行多步推理。目前学界对涌现现象的本质仍在讨论，一种解释是这些能力其实是渐变的，只是我们的评估指标（准确率）是离散的，看起来像突变。"

### GPT-3的革命性突破：In-Context Learning

```
不需要微调！只需要在输入中给几个例子，模型就能理解新任务：

Zero-shot（零样本）：
  Q: 把这句话翻译成英文："今天天气真好"

Few-shot（少样本，给几个例子）：
  中文: 你好         英文: Hello

  中文: 再见         英文: Goodbye
  中文: 今天天气真好   英文: ???
  A: The weather is really nice today.
```

## 5.3 预训练 Pre-training

### 🧠 一句话理解
> 在海量文本上，用"预测下一个词"这个任务，让模型学习语言的所有知识。

### 训练目标：自回归语言模型

```
给定序列 x₁, x₂, ..., xₙ，最大化：

P(x₁, x₂, ..., xₙ) = Π P(xₜ | x₁, ..., xₜ₋₁)

简单说：给你前面所有的词，预测下一个词
每一步都有"正确答案"（下一个词就是答案），不需要人工标注！
```

### 预训练数据

```
LLaMA-2训练数据（举例）：
├── CommonCrawl（网页爬取）    45%
├── C4（清洗后的网页）          27%
├── GitHub代码                  4.5%
├── Wikipedia                   4.5%
├── 书籍                        4.5%
├── ArXiv论文                   2.5%
└── Stack Exchange              2%
```

### 数据清洗的重要性

```
原始网页数据 → 去重 → 语言过滤 → 质量过滤 → 有害内容过滤 → 训练数据

关键步骤：
1. 去重（minhash等算法）：相同/相似内容只保留一份
2. 质量过滤：用小模型打分，过滤低质量文本
3. 有害内容过滤：去除色情、暴力、歧视内容
4. 数据配比：不同来源的数据按比例混合
```

## 5.4 监督微调 SFT (Supervised Fine-Tuning)

### 🧠 一句话理解
> 预训练模型只会"续写文本"，SFT让它学会"按照指令对话"——从语言模型变成对话助手。

### 🍎 类比
> 预训练后的GPT像一个"读了所有书的人"，但不知道如何回答问题。
> SFT像是给他培训"如何做客服"——用问答对话样本教他如何接受指令并给出有帮助的回答。

### SFT的数据格式

```
数据格式（指令-回答对）：

[
  {
    "instruction": "用一句话解释什么是机器学习",
    "response": "机器学习是让计算机从数据中自动学习规律的技术，无需人工编写规则。"
  },
  {
    "instruction": "写一首关于秋天的诗",
    "response": "落叶纷纷秋风起，..."
  }
]

关键：指令要多样（不同任务、不同风格）
     回答要高质量（专家撰写或精心筛选）
```

### SFT的效果
> 即使只用几千条高质量指令数据进行SFT，也能让模型从"文本补全"变成"能用的对话助手"。
> 著名实验：Stanford Alpaca用52K条数据微调LLaMA，效果接近GPT-3.5。

## 5.5 RLHF（人类反馈强化学习）

### 🧠 一句话理解
> 让人类为模型的不同回答打分，训练一个"奖励模型"来模拟人类偏好，然后用强化学习让LLM生成更符合人类期望的回答。

### 🍎 类比：培训员工
> SFT：给员工看好的回答样本（示范学习）
> RLHF：让员工给出多个回答，由客户打分，然后根据打分调整员工行为（反馈学习）

### RLHF三阶段

```
阶段一：SFT（已完成）
  → 得到基础对话模型

阶段二：训练奖励模型（Reward Model, RM）
  
  对同一个问题，让SFT模型生成多个回答：
  问：如何减肥？
  回答A："管住嘴迈开腿，坚持运动并控制热量摄入..."  （人类评分：4分）
  回答B："减肥最快的方法是不吃饭"                    （人类评分：1分）
  回答C："科学减肥需要注意以下几点..."               （人类评分：5分）
  
  用这些人类偏好数据训练奖励模型：
  RM(问题, 回答) → 分数（预测人类会给多少分）

阶段三：PPO强化学习优化

  用奖励模型作为"评委"，优化LLM使其输出高分回答：
  
  LLM生成回答 → RM打分 → PPO算法更新LLM参数
  
  + KL散度约束：防止模型过度偏离SFT模型（避免"奖励作弊"）
```

### 为什么需要RLHF？只有SFT不够吗？

```
SFT的问题：
  - 模拟"人类写的好答案"，但无法优化"哪个答案更好"
  - 对有害内容、幻觉等问题处理不够
  
RLHF解决了：
  - 对齐（Alignment）：让模型行为符合人类价值观
  - Helpful（有帮助）：回答真正解决用户问题
  - Harmless（无害）：拒绝有害请求
  - Honest（诚实）：不编造信息
  
这就是"HHH原则"——Anthropic Claude的对齐目标
```

### DPO（直接偏好优化）——RLHF的替代方案

```
RLHF的问题：
  - 需要单独训练奖励模型（复杂）
  - PPO算法不稳定，超参难调

DPO（Direct Preference Optimization，2023）：
  - 直接用偏好数据优化LLM，不需要奖励模型
  - 数学上等价于RLHF，但更简单稳定
  - 已成为业界主流替代方案

数据格式：
  {
    "prompt": "如何减肥？",
    "chosen": "科学减肥需要...",   ← 人类偏好的回答
    "rejected": "不吃饭最快..."   ← 人类不喜欢的回答
  }
```

#### ❓ 面试常问
**Q：解释RLHF的完整流程？**
> 💡 "RLHF分三步：第一步SFT，用高质量指令数据微调预训练模型，让它学会基本的对话能力；第二步训练奖励模型，对同一问题生成多个回答，请人类按偏好排序，用排序数据训练RM来模拟人类评分；第三步用PPO强化学习优化LLM，以RM的打分作为奖励信号，同时加KL约束防止模型偏离太远。目标是让模型学会生成Helpful、Harmless、Honest的回答。"

**Q：DPO相比RLHF有什么优势？**
> 💡 "DPO直接用偏好数据（选择/拒绝对）端到端优化LLM，省去了单独训练奖励模型的步骤，理论上等价于RLHF但更简单、更稳定。主要优势是实现简单（只需一个交叉熵损失变体）、训练稳定、不需要PPO复杂的超参调节。缺点是需要偏好数据质量高，且灵活性略低。"

## 5.6 高效微调技术 / Parameter-Efficient Fine-Tuning

### 为什么需要高效微调？

```
问题：GPT-4级别模型参数量极大，全量微调需要：
  - 巨大显存（全参数梯度）
  - 海量存储（每个任务存一份完整模型）
  - 昂贵算力

解决方案：只微调一小部分参数，效果接近全量微调
```

---

### 5.6.1 LoRA（低秩适配）

#### 🧠 一句话理解
> 不直接修改原始大权重矩阵，而是在旁边"插入"两个小矩阵，只训练这两个小矩阵。

#### 🍎 类比
> 原始模型是一本大百科全书（不动它）
> LoRA是在每页旁边贴了一张便利贴（只写少量修改内容）
> 使用时：原内容 + 便利贴修改 = 最终输出

```
原始方式（全量微调）：
  输出 = W × 输入         W是大矩阵（如4096×4096 = 1600万参数）

LoRA方式：
  输出 = W × 输入 + (A × B) × 输入
               ↑     ↑
         冻结不训练  只训练A和B（小矩阵）
  
  W: 4096 × 4096 = 1600万参数（冻结）
  A: 4096 × r    ← r=8（秩，超参数）
  B: r × 4096
  A×B: 4096 × 4096，但参数量只有 2 × 4096 × 8 = 65536（少了245倍！）
  
  r越小，参数越少，但表达能力也越弱
  实践中 r=4~64，根据任务复杂度选择
```

#### LoRA的优势
- 训练参数只有全量的 0.1%~1%
- 原始模型权重不变，多个任务的LoRA权重可以动态切换
- 推理时可以合并：W_new = W + A×B，不增加推理延迟

---

### 5.6.2 QLoRA

#### 🧠 一句话理解
> LoRA + 量化（Quantization）= 4bit量化的大模型 + LoRA微调，使在消费级GPU（24GB显存）上微调65B模型成为可能。

```
普通LoRA：基础模型用float16（16bit），显存需求大
QLoRA：基础模型量化到4bit存储（NF4格式），显存减少75%
       计算时再反量化到float16，精度损失很小

实际效果：65B参数模型的微调从需要8×80GB GPU
                              缩减到只需1×48GB GPU
```

#### ❓ 面试常问
**Q：LoRA的原理是什么？为什么能减少训练参数？**
> 💡 "LoRA基于一个假设：预训练权重矩阵本身是满秩的，但在特定任务微调时，权重变化量ΔW的内在维度很低（低秩），可以用两个低秩矩阵A和B的乘积来近似。具体做法是冻结原始权重W，插入旁路矩阵ΔW=AB（A形状d×r，B形状r×d，r远小于d），只训练A和B。训练结束后可以将AB合并回W，不增加推理开销。典型地，r=8时参数量仅为全量微调的0.1%左右，但效果接近。"

## 5.7 推理优化 / Inference Optimization

### 5.7.1 量化 Quantization

```
问题：模型权重默认用float32（32bit）或float16（16bit）存储，显存占用大

量化：用更低精度存储权重
  float32 → int8（8bit）：显存减半，精度略降
  float32 → int4（4bit）：显存减为1/8，精度有一定损失

直觉：
  float32：记录到小数点后7位（很精确）
  int8：   只记录整数（范围-128到127）
  
  对权重来说：精确到小数点后7位 vs 只保留大致范围
             → 大部分情况下效果差不多，但显存大幅减少
```

### 5.7.2 KV Cache

#### 🧠 一句话理解
> 把已经计算过的Key和Value矩阵缓存下来，避免重复计算，大幅加速自回归生成。

```
自回归生成的问题：
  生成第100个词时，需要计算前99个词的K和V
  生成第101个词时，又要重新计算前100个词的K和V
  → 大量重复计算！

KV Cache解决方案：
  生成第t个词时，把K_t和V_t存储起来
  生成第t+1个词时，直接复用已存的K₁~K_t，只计算K_{t+1}
  
  代价：需要额外显存存储KV Cache
       序列越长，KV Cache越大（长文本推理的瓶颈）
```

### 5.7.3 常用推理框架

| 框架 | 特点 | 适用场景 |
|------|------|---------|
| **vLLM** | PagedAttention，高吞吐 | 服务器端部署 |
| **llama.cpp** | CPU推理，量化支持 | 本地运行 |
| **TensorRT-LLM** | NVIDIA GPU优化 | 生产部署 |
| **Ollama** | 一键本地运行 | 个人使用 |

## 5.8 Prompt Engineering

### 🧠 一句话理解
> Prompt Engineering是通过精心设计输入提示来最大化LLM的输出质量，无需修改模型参数。

---

### 5.8.1 基础技巧

**明确指令**：
```
❌ "写点关于机器学习的内容"
✅ "用通俗语言，用3个要点总结机器学习的核心概念，每点不超过50字，面向零基础读者"
```

**角色扮演**：
```
"你是一位有10年经验的机器学习工程师，正在向一位产品经理解释深度学习..."
```

**输出格式控制**：
```
"请用JSON格式输出，包含字段：name, description, example"
```

---

### 5.8.2 Chain-of-Thought（思维链）

#### 🧠 一句话理解
> 让模型"先思考，再回答"，对复杂推理任务效果显著提升。

```
标准Prompt（效果差）：
  Q: 13 × 7 + 5 = ?
  A: 90（错误）

CoT Prompt（效果好）：
  Q: 13 × 7 + 5 = ? 请一步步思考。
  A: 
     先计算13×7：13×7 = 91
     然后加5：91 + 5 = 96
     答案是96。（正确）

关键：只需加上"一步步思考"/"Let's think step by step"这类触发语
     复杂数学/逻辑问题提升显著
```

### 5.8.3 Few-Shot Prompting

```
在Prompt中给几个示例，让模型理解任务格式：

示例：
  输入：这个产品真的太好用了！
  情感：正面

  输入：完全是浪费钱。
  情感：负面

  输入：还行，也没什么特别的。
  情感：？

模型会根据示例格式预测最后一个的情感。
```

### 5.8.4 结构化输出

```
"请以以下JSON格式输出，不要包含其他内容：
{
  'sentiment': '正面/负面/中性',
  'confidence': 0到1之间的置信度,
  'keywords': ['关键词1', '关键词2']
}"
```

#### ❓ 面试常问
**Q：什么是Chain-of-Thought？为什么有效？**
> 💡 "CoT通过在Prompt中引导模型展示中间推理步骤，而不是直接给出答案。有效的原因有两点：第一，分步推理将复杂问题分解成小步骤，每步更容易正确；第二，生成中间步骤相当于给模型更多'计算token'，实验发现token数量与推理能力相关。CoT在数学、逻辑、代码等需要多步推理的任务上效果显著，在简单任务上作用不大。"

## 5.9 RAG 检索增强生成

### 🧠 一句话理解
> 不改变模型参数，而是在回答问题前先从外部知识库检索相关文档，把文档内容塞进Prompt中，让模型基于检索到的内容回答。

### 🍎 类比
> LLM像一个参加闭卷考试的学生（只能用大脑里的知识）。
> RAG让这个学生变成开卷考试（可以查资料），但资料需要自己找到相关的部分。

### 为什么需要RAG？

```
LLM的局限：
  1. 知识截止日期（训练完后就不更新了）
  2. 可能幻觉（编造不存在的信息）
  3. 无法访问私有知识（公司内部文档）

RAG的解决方案：
  检索 + 生成 = 实时的、准确的、私有知识的回答
```

### RAG完整流程

```
离线阶段（建立知识库）：

文档（PDF/Word/网页）
    ↓ 文本切分（Chunking）
文本块（Chunks，每块约500字）
    ↓ Embedding模型（把文字变成向量）
向量（每块 → 一个1536维向量）
    ↓ 存入向量数据库
向量数据库（Chroma/Faiss/Pinecone等）

---

在线阶段（问答）：

用户问题："公司的年假政策是什么？"
    ↓ Embedding
问题向量
    ↓ 相似度检索（余弦相似度）
Top-K相关文本块（最相关的3-5段）
    ↓ 拼接进Prompt
Prompt = "根据以下内容回答问题：\n[检索到的文本]\n\n问题：公司的年假政策是什么？"
    ↓ LLM
答案（基于检索内容，有据可查）
```

### Embedding 和向量相似度

```
Embedding模型把文本映射为向量：
  "苹果公司"  → [0.23, -0.15, 0.87, ...]  （1536维）
  "Apple Inc" → [0.24, -0.14, 0.85, ...]  （非常接近！）
  "吃苹果"    → [0.67,  0.23, 0.12, ...]  （不太一样）

余弦相似度：
  cos("苹果公司", "Apple Inc") ≈ 0.98  （高度相关）
  cos("苹果公司", "吃苹果")   ≈ 0.45  （不太相关）
```

#### ❓ 面试常问
**Q：RAG的基本流程是什么？相比直接微调有什么优缺点？**
> 💡 "RAG分为离线构建和在线检索两阶段。离线阶段将文档切块、用Embedding模型转为向量、存入向量数据库。在线阶段将用户问题转为向量、检索TopK相关文档块、拼接进Prompt让LLM生成回答。相比微调，RAG优点是：知识可实时更新（改文档库即可）、成本低（不需要训练）、可追溯来源。缺点是：检索质量上限受限于Embedding模型、处理需要深度推理的问题效果不如微调、检索到不相关内容会引入噪声。"

## 5.11 章节总结 / Chapter Summary

### LLM训练全流程速记

```
互联网文本（TB级）
    ↓ 数据清洗
训练数据
    ↓ 自回归预训练（预测下一个词）  ← 消耗99%算力
预训练模型（Base Model）
    ↓ SFT（指令微调）
对话模型（Chat Model）
    ↓ RLHF / DPO（价值观对齐）
对齐模型（Aligned Model）← 最终用户使用的版本
```

### 核心概念速查

| 概念 | 一句话 |
|------|-------|
| 规模定律 | 参数×数据×算力，三者需均衡 |
| 涌现能力 | 达到临界规模后突然出现的新能力 |
| SFT | 用指令数据让模型学会对话 |
| RLHF | 用人类偏好反馈对齐价值观 |
| DPO | RLHF的简化替代，更稳定 |
| LoRA | 低秩适配，高效微调 |
| QLoRA | 4bit量化+LoRA，消费级GPU可微调 |
| RAG | 检索增强，解决知识截止和幻觉 |
| KV Cache | 缓存注意力计算，加速推理 |
| CoT | 思维链，让模型展示推理过程 |
