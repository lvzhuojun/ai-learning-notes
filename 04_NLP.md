## 4.2 文本表示的进化史 / Text Representation Evolution

> **核心问题**：神经网络只能处理数字，如何把"文字"变成"数字"？

---

### 4.2.1 One-Hot 编码

#### 🍎 类比
> 假设词典里只有5个词：[猫, 狗, 鸟, 鱼, 虫]
> "猫" = [1, 0, 0, 0, 0]
> "狗" = [0, 1, 0, 0, 0]
> "鸟" = [0, 0, 1, 0, 0]

#### 致命缺陷
```
"猫" = [1,0,0,0,0]
"狗" = [0,1,0,0,0]

cos("猫","狗") = 0   ← 相似度为0
cos("猫","飞机") = 0  ← 相似度也为0
```
> **所有词之间距离相等**，无法表达"猫和狗都是宠物"这种语义关系。
> 而且词典很大时（几万词），向量极其稀疏，维度爆炸。

---

### 4.2.2 TF-IDF

#### 🧠 一句话理解
> 衡量一个词对一篇文档的重要性：出现频率高 + 在其他文档中少见 = 重要。

```
TF（词频）= 该词在本文档出现次数 / 文档总词数
IDF（逆文档频率）= log(文档总数 / 包含该词的文档数)

TF-IDF = TF × IDF

例：
"的" → TF很高，但每篇文档都有 → IDF很低 → TF-IDF低 → 不重要
"量子" → TF低，只有少数文档有 → IDF高 → TF-IDF高 → 很重要
```

> **局限**：仍无法捕捉词的语义关系，"开心"和"快乐"被视为完全不同的词。

---

### 4.2.3 Word2Vec（词向量的革命）

#### 🧠 一句话理解
> 把每个词映射成一个低维稠密向量，**语义相似的词，向量也相近**。

#### 🍎 "分布式假说"
> **"一个词的含义由其上下文决定。"**（Distributional Hypothesis）
> 经常出现在相似上下文中的词，含义也相似。
> "我今天吃了___" → "苹果"和"香蕉"都能填 → 两者语义相似

#### Word2Vec的神奇性质

```
词向量可以做算术！

"国王" - "男人" + "女人" ≈ "女王"
"巴黎" - "法国" + "中国" ≈ "北京"
"走路" - "走" + "游泳" ≈ "游"

语义关系被编码进了向量的方向！
```

#### 两种训练方式

**CBOW（Continuous Bag of Words）**：
```
上下文词 → 预测中心词

"我 今天 ___ 了 苹果"  →  预测"吃"
```

**Skip-gram**：
```
中心词 → 预测上下文词

"吃"  →  预测周围的词（我、今天、了、苹果）
```

#### Word2Vec的局限
> **一词一义**：无法处理多义词
> "苹果"这个词无论在"吃苹果"还是"苹果公司"中，都是同一个向量
> → 解决方案：**上下文相关的词向量** → ELMo → Transformer

---

### 4.2.4 GloVe 和 FastText（了解即可）

| 模型 | 核心思想 | 特点 |
|------|---------|------|
| **GloVe** | 利用全局词共现统计 | 比Word2Vec训练更稳定 |
| **FastText** | 词 = 字符n-gram的组合 | 能处理未见过的词（OOV） |

## 4.3 语言模型 Language Model

### 🧠 一句话理解
> 语言模型估计一段文字出现的概率，或者给定前文预测下一个词。

```
P("今天天气很好") = ?
P("天气好今天很") = ?   ← 比上面概率低得多

给定 "今天天气很___" → 预测下一个词："好"(0.6) "差"(0.2) "糟"(0.1) ...
```

> **这个"预测下一个词"的任务，是GPT系列所有能力的根基！**（第5章详述）

### N-gram模型（统计语言模型）

```
Bigram（2-gram）：只看前1个词来预测下一个词
  P(好 | 很) = 在语料中"很好"出现次数 / "很"出现次数

Trigram（3-gram）：看前2个词
  P(好 | 天气很) = ...

缺陷：窗口太小，无法利用长距离上下文
```

## 4.4 Seq2Seq 与注意力机制 / Seq2Seq & Attention

### 4.4.1 Seq2Seq 架构（机器翻译的基础）

#### 🧠 一句话理解
> 先用编码器把整个输入序列"压缩"成一个向量，再用解码器从这个向量"解压"出输出序列。

```
机器翻译：「今天天气很好」→「The weather is nice today」

编码器（Encoder）            解码器（Decoder）
┌─────────────────────┐     ┌─────────────────────────┐
│今 → 天 → 气 → 很 → 好│─→   │→The →weather →is →nice →today│
│       [LSTM]        │  ↑  │          [LSTM]               │
└─────────────────────┘  │  └─────────────────────────────┘
                       语义向量
                     （信息瓶颈！）
```

#### 致命缺陷：信息瓶颈
> 不管输入多长，都要压缩成**一个固定大小的向量**——这个"上下文向量"装不下长句子的全部信息，导致翻译质量下降。

---

### 4.4.2 注意力机制 Attention Mechanism

#### 🧠 一句话理解
> 解码器在生成每个输出词时，动态"关注"编码器中与当前输出最相关的输入位置，而不是只用一个固定向量。

#### 🍎 类比：翻译时查原文
> 你在翻译"今天天气很好"时，翻译"today"时你会重新看"今天"这个词；翻译"weather"时你会看"天气"。
> 注意力机制让模型做同样的事——**翻译每个词时，动态决定要看原文的哪里**。

```
没有注意力（Seq2Seq）：
  今天天气很好 → [一个向量] → The weather is nice today
                   ↑ 信息瓶颈

有注意力（Attention Seq2Seq）：
  今天 天气  很  好
   ↑    ↑         ← 生成"today"时，主要关注"今天"
        ↑    ↑    ← 生成"nice"时，主要关注"天气""好"
  
  每个输出位置都有一个"注意力分布"，告诉模型该看哪里
```

#### 注意力的计算：Q、K、V

> 注意力机制的核心可以理解成一个"**查字典**"的过程：

```
Q（Query，查询）：我想找什么？     → 当前解码位置的"问题"
K（Key，键）：   字典里有什么词条？→ 编码器每个位置的"标签"
V（Value，值）：  词条对应的内容？  → 编码器每个位置的"内容"

步骤：
1. 计算Q和每个K的相似度 → 注意力分数（scores）
2. 用softmax归一化 → 注意力权重（0~1的概率分布）
3. 用权重对V加权求和 → 输出

公式：Attention(Q, K, V) = softmax(QKᵀ / √d_k) × V
                                        ↑
                             缩放因子，防止点积过大
```

## 4.5 Transformer 架构（核心！）

### 🧠 一句话理解
> Transformer完全用注意力机制代替了RNN，实现了**并行计算**和**捕捉任意长距离依赖**，是现代NLP和LLM的基础。

### 来自论文：「Attention Is All You Need」（2017，Vaswani et al.）
> 这篇论文是NLP历史上最重要的论文之一，标题的意思是："只需要注意力机制就够了（不需要RNN）"

---

### 4.5.1 为什么要抛弃RNN？

| 问题 | RNN/LSTM | Transformer |
|------|---------|-------------|
| 长程依赖 | 容易遗忘（梯度消失）| 任意距离的词直接相互关注 |
| 并行计算 | 必须顺序处理（t时刻依赖t-1）| 所有位置同时计算（高效！）|
| 训练速度 | 慢 | 快 |

---

### 4.5.2 自注意力 Self-Attention

#### 🧠 一句话理解
> 序列中的**每个词**都去关注**同一序列中的所有其他词**，学习词与词之间的关系。

#### 🍎 类比：句子内部的"社交网络"
> "苹果公司发布了新手机"
> "苹果"这个词需要知道旁边是"公司"（而不是"吃苹果"），才能被正确理解。
> Self-Attention让每个词都能"看到"句子中的所有其他词，动态决定"我应该最关注谁"。

#### 计算过程

```
输入句子："The cat sat on the mat"
每个词先映射成三个向量：Q（查询）、K（键）、V（值）

以"cat"为例：
  Q_cat = W_Q × embedding_cat    ← "cat这个词，我想找什么信息？"
  K_cat = W_K × embedding_cat    ← "cat这个词，我能提供什么信息？"
  V_cat = W_V × embedding_cat    ← "cat这个词，我的实际内容是？"

计算"cat"对所有词的注意力：
  score(cat, The) = Q_cat · K_The   = 2.1
  score(cat, cat) = Q_cat · K_cat   = 9.8  ← 自己对自己注意力最高（但未必）
  score(cat, sat) = Q_cat · K_sat   = 3.5
  score(cat, on)  = Q_cat · K_on    = 0.2
  ...

softmax归一化：[0.02, 0.75, 0.20, 0.01, ...] ← 注意力权重，和为1

输出：output_cat = 0.02×V_The + 0.75×V_cat + 0.20×V_sat + 0.01×V_on + ...
               ← "cat"的新表示，融合了它最关注的词的信息
```

### 4.5.3 多头注意力 Multi-Head Attention

#### 🧠 一句话理解
> 并行运行多个自注意力头，每个头关注不同方面的信息，然后把结果拼接起来。

#### 🍎 类比：多个视角
> 看一段文字，有人关注**语法关系**（主语和谓语），有人关注**指代关系**（"它"指代什么），有人关注**语义关系**（哪些词意思相近）。
> 多头注意力让模型同时从多个视角理解文本。

```
Multi-Head Attention（h=8个头）：

输入
 ├──→ Head₁（W_Q₁, W_K₁, W_V₁）→ output₁
 ├──→ Head₂（W_Q₂, W_K₂, W_V₂）→ output₂
 ├──→ Head₃（W_Q₃, W_K₃, W_V₃）→ output₃
 ...
 └──→ Head₈（W_Q₈, W_K₈, W_V₈）→ output₈

Concat(output₁,...,output₈) → 线性变换 → 最终输出
```

---

### 4.5.4 位置编码 Positional Encoding

#### 🧠 一句话理解
> Self-Attention本身没有顺序感（"猫追狗"和"狗追猫"会得到相同的表示），需要额外注入位置信息。

#### 🍎 类比
> Self-Attention就像把一副牌摊在桌上同时看（没有顺序），位置编码就是在每张牌上标注"这是第1张/第2张/第3张..."。

```
位置编码公式（原始Transformer使用正弦/余弦函数）：
  PE(pos, 2i)   = sin(pos / 10000^(2i/d_model))
  PE(pos, 2i+1) = cos(pos / 10000^(2i/d_model))

最终输入 = 词嵌入 + 位置编码
```

### 4.5.5 完整Transformer架构

```
原始Transformer（用于机器翻译）：

     输出概率分布
          ↑
    ┌─────────────┐
    │  线性 + Softmax │
    └─────────────┘
          ↑
┌─────────────────────┐
│      解码器 Decoder   │   ← 6层堆叠
│                     │
│  [多头注意力] ← 编码器输出  │   ← 交叉注意力（Cross-Attention）
│  [自注意力（带掩码）]  │   ← Masked Self-Attention（不能看未来词）
│  [前馈网络 FFN]       │
│  [残差连接 + LayerNorm]│
└─────────────────────┘
          ↑ 编码器输出
┌─────────────────────┐
│      编码器 Encoder   │   ← 6层堆叠
│                     │
│  [多头自注意力]        │
│  [前馈网络 FFN]       │
│  [残差连接 + LayerNorm]│
└─────────────────────┘
          ↑
    输入词嵌入 + 位置编码

前馈网络（FFN）：
  FFN(x) = max(0, xW₁ + b₁)W₂ + b₂
  ← 两层全连接，每个位置独立计算，进一步转换特征
```

#### Layer Normalization（区别于Batch Norm）

```
BatchNorm：对一个batch内所有样本的同一特征做归一化（跨样本）
LayerNorm：对同一样本的所有特征做归一化（跨特征）

Transformer用LayerNorm的原因：
  序列长度可变，BatchNorm在可变长序列上效果差
  LayerNorm不依赖batch大小，更稳定
```

### 4.5.6 Transformer 关键设计总结

| 组件 | 作用 | 为什么重要 |
|------|------|---------|
| Self-Attention | 建模词间关系 | 任意距离依赖，并行计算 |
| Multi-Head | 多视角关注 | 捕捉不同类型的语言关系 |
| 位置编码 | 注入顺序信息 | 弥补注意力无序的缺陷 |
| 残差连接 | 梯度通道 | 允许训练深层网络 |
| Layer Norm | 稳定训练 | 归一化激活分布 |
| FFN | 特征变换 | 位置独立的非线性变换 |

#### ❓ 面试常问（高频！）

**Q：Transformer中Self-Attention的计算复杂度是多少？有什么问题？**
> 💡 "Self-Attention的时间和空间复杂度是O(n²×d)，其中n是序列长度，d是维度。因为每个位置都要和其他所有位置计算相似度，形成n×n的注意力矩阵。这导致当序列很长时（如处理整本书）计算代价极高，这是LLM处理长文本的核心瓶颈，也是Flash Attention、Linear Attention等改进工作的出发点。"

**Q：为什么要除以√d_k？**
> 💡 "当向量维度d_k较大时，Q和K的点积结果方差也变大，会让softmax的输入进入梯度极小的饱和区，导致梯度消失。除以√d_k相当于对点积做缩放，使其方差恢复到1，让softmax在梯度较大的区域工作，训练更稳定。"

**Q：Transformer和RNN相比有什么优缺点？**
> 💡 "优点：并行计算（RNN必须顺序计算）、能捕捉任意长距离依赖（不受梯度消失限制）、训练更快。缺点：Self-Attention计算复杂度O(n²)，处理超长序列代价高；没有内置的位置感知，需要额外的位置编码；对小数据集可能过拟合（参数量大）。"

**Q：Encoder和Decoder在Transformer中的区别？**
> 💡 "Encoder用双向Self-Attention，每个位置可以看到整个输入序列（适合理解任务）。Decoder有两种注意力：一是Masked Self-Attention（只能看到当前位置之前的词，防止'偷看'未来）；二是Cross-Attention（关注Encoder的输出）。BERT用纯Encoder，GPT用纯Decoder，T5等模型用完整Encoder-Decoder。"

## 4.6 预训练语言模型 / Pre-trained Language Models

### 🧠 一句话理解
> 在海量文本上预先训练好一个通用模型，然后在特定任务上稍加微调，效果远超从头训练。

### 🍎 类比：通才 vs 专才
> 以前训练模型像培养"专才"：每个任务单独培训一个模型。
> 预训练就像先让人读了10年书（学习通用语言知识），然后再去各个岗位工作，只需要短暂培训就能上手。

---

### 4.6.1 BERT（双向编码器）

**全称**：Bidirectional Encoder Representations from Transformers（2018，Google）

#### 核心创新：双向上下文

```
GPT（单向，自左向右）：
  "今天 天气 ___ 好"  → 只能看左边的词来预测

BERT（双向）：
  "今天 天气 [MASK] 好" → 能同时看左边和右边的词来预测[MASK]
                           ↑ 这叫 Masked Language Model (MLM)
```

#### BERT的两个预训练任务

**任务1：Masked Language Model (MLM)**
```
输入：  "今天 [MASK] 天气 很 好"
目标：  预测[MASK]位置的词（正确答案："的"）

随机遮住15%的词，让模型预测
→ 强迫模型理解双向上下文
```

**任务2：Next Sentence Prediction (NSP)**
```
输入A："小明今天去了图书馆"
输入B："他借了三本书"  → 是A的下一句？是 ✓

输入A："小明今天去了图书馆"
输入B："天空中有朵白云"  → 是A的下一句？否 ✗

→ 帮助模型理解句间关系（用于问答、推理等任务）
```

#### BERT的特征：上下文词向量
> "苹果"在"吃苹果"和"苹果公司"中的BERT表示**完全不同**！
> BERT解决了Word2Vec"一词一义"的问题。

---

### 4.6.2 GPT 系列（单向解码器）

**全称**：Generative Pre-trained Transformer（OpenAI）

#### 核心：语言建模（预测下一个词）

```
预训练任务：给定前文，预测下一个词

"今天 天气" → 预测 "很"
"今天 天气 很" → 预测 "好"
"今天 天气 很 好" → 预测 "。"

这是一个自监督任务：训练数据不需要人工标注！
只要有文本就可以训练！
```

#### GPT vs BERT

| 对比 | BERT | GPT |
|------|------|-----|
| 方向 | 双向（Encoder）| 单向从左到右（Decoder）|
| 预训练任务 | 填空（MLM）| 预测下一个词 |
| 适合任务 | 理解（分类、NER）| 生成（对话、续写）|
| 代表应用 | 文本分类、问答 | ChatGPT、代码生成 |

---

### 4.6.3 预训练→微调范式

```
阶段1：预训练（Pre-training）
  大量无标注文本（整个互联网）
  自监督学习（MLM/CLM）
  → 得到通用语言模型

阶段2：微调（Fine-tuning）
  少量有标注的下游任务数据
  监督学习（交叉熵损失）
  → 得到专用模型

对比：
  以前：每个任务从头训练，需要大量标注数据
  现在：预训练一次，微调可以很少的数据
```

## 4.7 章节总结 / Chapter Summary

### NLP发展时间线

```
1950s  →  2003  →  2013     →  2014      →  2017        →  2018    →  2020+
图灵测试   N-gram  Word2Vec    Seq2Seq     Transformer     BERT      GPT-3
规则系统   语言模型 词嵌入       注意力机制   "注意力即一切"   预训练     大模型时代
                              提出
```

### 技术演进的核心矛盾

```
核心问题：如何表示词的含义？
          ↓
One-Hot → 无语义关系
          ↓
Word2Vec → 有语义关系，但一词一义
          ↓
LSTM+Attention → 上下文相关，但并行性差
          ↓
Transformer → 上下文相关 + 并行 + 可扩展
```
