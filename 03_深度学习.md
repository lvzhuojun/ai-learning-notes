## 3.1 神经网络基础 / Neural Network Fundamentals

### 🧠 一句话理解
> 神经网络是受大脑启发设计的计算模型：由大量简单的"神经元"连接而成，通过层层变换从原始数据中提取复杂特征。

### 🍎 类比：从生物到机器
```
生物神经元：                   人工神经元：
                              
  输入信号₁ →                 x₁ → (×w₁)
  输入信号₂ → [细胞体] → 输出   x₂ → (×w₂) → [求和+偏置] → [激活函数] → 输出
  输入信号₃ →                 x₃ → (×w₃)
  
  突触权重决定信号强弱           权重 w 决定输入的重要程度
  超过阈值才发射信号             激活函数决定是否"激活"
```

---

### 3.1.1 感知机 Perceptron

#### 最简单的神经网络（单个神经元）

```
输入：x₁, x₂, x₃
权重：w₁, w₂, w₃
偏置：b

计算：z = w₁x₁ + w₂x₂ + w₃x₃ + b
输出：y = 激活函数(z)
```

#### 感知机的局限
> 单层感知机只能解决**线性可分**问题，对于XOR（异或）这类非线性问题完全无能为力。
> 解决方案：**多层感知机（MLP）**

---

### 3.1.2 多层感知机 MLP（全连接网络）

```
                 隐藏层₁    隐藏层₂
输入层           (4个神经元) (4个神经元)  输出层

x₁ ──────┬──── ○ ──┬──── ○ ──┐
          ├──── ○ ──┼──── ○ ──┤
x₂ ──────┼──── ○ ──┼──── ○ ──┼──── ŷ（预测值）
          ├──── ○ ──┼──── ○ ──┤
x₃ ──────┘──── ○ ──┘──── ○ ──┘

每个连接都有一个权重 w
每一层都做：线性变换 → 激活函数
```

#### 为什么需要多层？
> 每层学习不同层次的抽象特征：
```
以图像识别为例：
第1层 → 边缘、颜色
第2层 → 形状（圆形、直线）
第3层 → 零件（眼睛、嘴巴）
第4层 → 整体概念（人脸）
```

---

### 3.1.3 激活函数 Activation Functions

#### 🧠 一句话理解
> 激活函数引入**非线性**，没有它，无论多少层网络叠加，效果都等同于一层线性变换。

#### 🍎 类比
> 没有激活函数的神经网络，就像无数张透明胶片叠在一起——无论叠多少层，看上去还是一张透明胶片。激活函数让每层胶片变成"折叠的"，这样叠起来才有意义。

#### 常见激活函数对比

**Sigmoid**
```
公式：σ(x) = 1 / (1 + e⁻ˣ)
输出范围：(0, 1)

图形：
  1 │        ╭──────
    │      ╭╯
0.5 │    ──┤
    │  ╭╯
  0 │╭╯
    └────────────────→ x

优点：输出可解释为概率
缺点：梯度消失（两端导数趋近于0），计算慢
用途：二分类输出层
```

**Tanh**
```
公式：tanh(x) = (eˣ - e⁻ˣ) / (eˣ + e⁻ˣ)
输出范围：(-1, 1)

比Sigmoid好：输出以0为中心，收敛更快
仍有缺点：梯度消失问题依然存在
```

**ReLU（最常用）**
```
公式：ReLU(x) = max(0, x)
输出范围：[0, +∞)

图形：
    │     ╱
    │    ╱
    │   ╱
  0 │──╱
    └────────────────→ x
    负数→0  正数→原值

优点：计算极简单，有效缓解梯度消失，收敛快
缺点：负数输入时梯度为0（"神经元死亡"问题）
用途：隐藏层的默认选择
```

**Leaky ReLU / ELU**
```
Leaky ReLU(x) = x     (x > 0)
              = αx    (x ≤ 0, α通常=0.01)

解决了ReLU的"死亡"问题：负数区域有小的负斜率
```

**Softmax（多分类输出专用）**
```
公式：softmax(xᵢ) = eˣⁱ / Σeˣʲ

将一组数值转换为概率分布，所有输出之和为1

例：[2.0, 1.0, 0.1] → [0.659, 0.242, 0.099]
                         ↑ 第1类概率最高
```

#### ❓ 面试常问
**Q：为什么ReLU比Sigmoid更常用？**
> 💡 "三个原因：第一，ReLU计算极简（只需比较大小），反向传播梯度也是常数1或0，不像Sigmoid的梯度在两端趋近于0（梯度消失问题）；第二，ReLU训练收敛更快；第三，ReLU产生稀疏激活（负数输出为0），相当于正则化。缺点是可能出现'神经元死亡'（Dead ReLU），可用Leaky ReLU缓解。"

**Q：为什么需要激活函数？**
> 💡 "激活函数引入非线性。如果没有激活函数，无论网络多深，多层线性变换叠加仍然是线性变换，表达能力与单层等价，无法学习复杂的非线性规律。有了非线性激活函数，深层网络才能学习任意复杂的函数映射（通用近似定理）。"

## 3.2 训练机制 / Training Mechanism

### 3.2.1 正向传播 Forward Propagation

```
输入数据 → 逐层计算（线性变换+激活函数）→ 得到预测值 → 计算损失
```

### 3.2.2 反向传播 Backpropagation

#### 🧠 一句话理解
> 反向传播是用**链式法则**，从输出层往回计算每个参数对损失的"责任"（梯度），然后更新参数。

#### 🍎 类比
> 就像追责：最终结果（考试成绩差）→ 倒推是哪门课没学好（哪层参数有问题）→ 再倒推是哪个知识点没掌握（哪个参数的梯度大）→ 针对性补强（更新参数）。

#### 链式法则 Chain Rule
```
损失L对权重w₁的梯度：

∂L/∂w₁ = ∂L/∂ŷ × ∂ŷ/∂h₂ × ∂h₂/∂h₁ × ∂h₁/∂w₁

         ↑每一步都是相邻层的偏导数，链式相乘
```

#### 梯度下降更新规则
```
w_new = w_old - 学习率 × ∂L/∂w

学习率（Learning Rate, lr）控制每次更新的步长：
  lr太大 → 跳过最优解，震荡
  lr太小 → 收敛极慢
  
通常设置：0.001 ~ 0.1，需要实验调整
```

---

### 3.2.3 梯度消失与梯度爆炸

#### 梯度消失 Vanishing Gradient

#### 🧠 一句话理解
> 在深层网络中，梯度从输出层往回传递时，每层都乘以一个小于1的数，经过很多层后梯度趋近于0，导致浅层参数几乎不更新。

#### 🍎 类比
> 传话游戏：老师的话传过10个人后，最开始的学生已经几乎听不到消息了。

```
Sigmoid梯度最大值 = 0.25

在20层网络中：
  梯度 = 0.25²⁰ ≈ 0.000000000091 ≈ 0  → 浅层完全学不到东西
```

#### 解决方案
- 使用ReLU代替Sigmoid（梯度为1，不会缩小）
- 残差连接（ResNet）：提供梯度"高速公路"
- Batch Normalization：归一化激活值
- 梯度裁剪（用于RNN）

#### 梯度爆炸 Exploding Gradient
> 与梯度消失相反：梯度越来越大，参数更新剧烈，训练不稳定。
> **解决**：梯度裁剪（Gradient Clipping）——设置梯度上限

---

### 3.2.4 优化器 Optimizers

#### 🧠 一句话理解
> 优化器决定"如何用梯度更新参数"，是训练速度和质量的关键。

#### 进化历程

**SGD（随机梯度下降）**
```
每次只用一个样本（或小批量）计算梯度并更新

优点：每步计算快，可跳出局部最优
缺点：震荡严重，收敛不稳定
```

**Momentum（动量）**
```
引入"惯性"：更新方向 = 当前梯度 + 上一步方向的一部分

就像滚雪球：之前的方向有一定惯性，不会急刹车
解决了SGD震荡问题
```

**Adam（最常用）= Momentum + 自适应学习率**
```
两个改进：
1. 记录梯度的一阶矩（方向）→ 类似Momentum
2. 记录梯度的二阶矩（大小）→ 为每个参数自适应调整学习率

效果：大梯度的参数学习率变小，小梯度的参数学习率变大
     → 自动平衡不同参数的更新速度
```

#### 优化器对比

| 优化器 | 特点 | 推荐场景 |
|-------|------|---------|
| SGD | 简单，需要调学习率 | 图像分类（精调时） |
| SGD+Momentum | 收敛更稳定 | 通用 |
| Adam | 自适应，调参简单 | **默认首选** |
| AdamW | Adam + 权重衰减修正 | Transformer训练 |
| RMSprop | Adam的前身 | RNN |

#### ❓ 面试常问
**Q：Adam优化器的原理？为什么比SGD好？**
> 💡 "Adam结合了Momentum和RMSprop的优点。它维护两个动量：梯度的一阶矩（均值，决定更新方向，类似Momentum）和梯度的二阶矩（未中心化方差，决定每个参数的学习率大小）。效果是为频繁更新的参数自动降低学习率，为稀疏梯度的参数增大学习率，不需要手动调学习率，训练更稳定。"

## 3.3 正则化技术 / Regularization in Deep Learning

### 3.3.1 Dropout

#### 🧠 一句话理解
> 训练时随机"关掉"一部分神经元（设为0），强迫网络不依赖特定神经元，学习更鲁棒的特征。

#### 🍎 类比
> 像一个团队，随机让部分成员"请假"，剩下的人要顶上去——久而久之，每个人都学会了多面手，团队更健壮。

```
正常网络：
  ○ → ○ → ○
  ○ → ○ → ○
  ○ → ○ → ○

Dropout（p=0.5，50%的神经元被关掉）：
  ○ → × → ○      ← ×表示该神经元被关掉（输出置为0）
  × → ○ → ×
  ○ → ○ → ○
```

#### 关键细节
- **只在训练时用**，测试时所有神经元都激活（但输出要乘以保留概率p来补偿）
- 常见Dropout率：0.5（隐藏层）, 0.1~0.2（输入层）
- 相当于训练了多个不同的子网络，测试时取平均（集成效果）

#### ❓ 面试常问
**Q：Dropout的原理和作用？训练和测试阶段有什么区别？**
> 💡 "Dropout在训练时以概率p随机将神经元置零，强迫网络学习冗余表示，防止神经元之间的协同依赖，起到正则化效果。测试时不使用Dropout，所有神经元都激活，但需要将输出乘以保留概率(1-p)，以保持期望值一致（也可以用inverted dropout，在训练时就除以保留概率）。"

---

### 3.3.2 批归一化 Batch Normalization

#### 🧠 一句话理解
> 对每层的激活值做归一化，使其均值为0、方差为1，让训练更稳定、更快。

#### 为什么需要？——内部协变量偏移问题
```
问题：每层的输入分布随着前层参数更新而不断变化
      → 每层都要"追赶"变化的输入分布
      → 训练慢，需要非常小的学习率

BatchNorm：强制每层输入在每个mini-batch内保持稳定分布
           → 可以用更大的学习率，训练更快
```

#### 计算过程
```
给定一个mini-batch的激活值 {x₁, x₂, ..., xₘ}：

1. 计算均值：μ = (1/m) × Σxᵢ
2. 计算方差：σ² = (1/m) × Σ(xᵢ - μ)²
3. 归一化：  x̂ᵢ = (xᵢ - μ) / √(σ² + ε)
4. 缩放平移：yᵢ = γx̂ᵢ + β    ← γ和β是可学习参数，让网络自己决定要多"归一"
```

#### ❓ 面试常问
**Q：Batch Normalization有什么作用？训练和推理时有什么区别？**
> 💡 "BatchNorm有三个作用：加速训练（允许更大学习率）、减少对参数初始化的敏感性、有轻微正则化效果（因为mini-batch的均值/方差引入了噪声）。训练时用当前mini-batch的均值和方差；推理时因为可能只有单个样本，改用训练过程中累计的滑动平均均值和方差（moving average），确保结果稳定。"

## 3.4 卷积神经网络 CNN

### 🧠 一句话理解
> CNN通过"卷积核"自动学习图像中的局部特征（边缘→纹理→形状→物体），是图像处理的核心架构。

### 🍎 类比：手电筒扫描
> 用一个小手电筒（卷积核/滤波器）在图片上滑动扫描，每次只看一小块区域，记录"这里有没有我在找的特征"。

### 3.4.1 卷积操作

```
输入图像（5×5）：        卷积核（3×3）：
┌─────────────┐         ┌───────┐
│ 1  2  3  0  1│         │ 1  0  1│
│ 0  1  2  3  1│    ×    │ 0  1  0│   →   特征图（Feature Map）
│ 2  0  1  2  0│         │ 1  0  1│
│ 1  2  3  1  2│         └───────┘
│ 0  1  2  0  1│
└─────────────┘

计算过程（只看左上角3×3块）：
1×1 + 2×0 + 3×1
0×0 + 1×1 + 2×0  = 1+3+0+1+1+0 = 10
2×1 + 0×0 + 1×1
                 → 特征图该位置值为10

滑动步长（Stride）：每次移动几格
填充（Padding）：边缘补0，保持输出尺寸
```

### 3.4.2 为什么CNN比全连接更适合图像？

| 特性 | 全连接 FC | 卷积 CNN |
|------|----------|---------|
| 参数数量 | 极多（每个像素都连接）| 极少（卷积核共享） |
| 位置不变性 | 无（位置变了就不认识）| 有（特征在哪里都能识别）|
| 感受野 | 整张图 | 局部区域（层层扩大）|

**参数共享（权重共享）**：
> 同一个卷积核扫描整张图的不同位置——"检测竖线"的卷积核，在图像左上角检测竖线和在右下角检测竖线，用的是同一个核，大幅减少参数量。

### 3.4.3 池化层 Pooling

```
作用：对特征图进行下采样，减少参数量，提高平移不变性

最大池化（Max Pooling，最常用）：
  输入（4×4）：          输出（2×2）：
  ┌──────────┐          ┌──────┐
  │ 1  3  2  4│          │  4  4│
  │ 2  4  3  1│   →      │  4  3│
  │1  2  4  2│
  │2  3  3  1│
  └──────────┘
  每个2×2区域取最大值
```

### 3.4.4 经典CNN架构演进

```
LeNet(1998) → AlexNet(2012) → VGG(2014) → ResNet(2015) → ...

       ↑           ↑              ↑            ↑
  手写数字     ImageNet大赛     更深更宽    残差连接解决
  识别先驱     深度学习爆发     验证深度    梯度消失
              的起点          有效
```

**ResNet（残差网络）的关键创新**：

```
普通网络：
  输入x → [卷积层] → [卷积层] → 输出H(x)

残差网络：
           ┌─────────────────────┐
  输入x → [卷积层] → [卷积层] → (+) → 输出H(x) = F(x) + x
           └────── 跳跃连接 ──────┘
                   (直接相加)

关键：跳跃连接让梯度可以"跳过"这些层直接传到前面
     解决了深层网络的梯度消失问题，使训练100+层成为可能
```

#### ❓ 面试常问
**Q：ResNet为什么有效？残差连接解决了什么问题？**
> 💡 "ResNet引入了跳跃连接（skip connection），将输入直接加到若干层之后的输出上。这解决了两个问题：第一，梯度可以通过跳跃连接直接反传到浅层，缓解了梯度消失；第二，如果某些层的变换是多余的，网络可以学习将这些层的权重趋近于零（此时输出≈输入），即学习'恒等映射'比学习零映射更容易，降低了优化难度。"

## 3.5 循环神经网络 RNN / LSTM / GRU

### 🧠 一句话理解
> RNN是处理**序列数据**的网络，通过"记忆"之前的信息来理解当前输入。

### 🍎 类比
> 读一本书，理解每句话都需要联系前面的上下文——RNN就是有"短期记忆"的网络。

### 3.5.1 RNN基础

```
时间步展开：

x₁ → [RNN] → h₁
       ↑↓
x₂ → [RNN] → h₂      hₜ = f(xₜ, hₜ₋₁)
       ↑↓                        ↑      ↑
x₃ → [RNN] → h₃            当前输入  上一步隐状态
       ↑↓
      ...

隐状态 h 就是网络的"记忆"
```

**RNN的致命缺陷：长程依赖问题**
> 序列很长时，早期信息在隐状态中被稀释，网络"记不住"很久以前的内容。
> 数学本质：仍是梯度消失问题（在时间维度上）

### 3.5.2 LSTM（长短期记忆网络）

#### 🧠 一句话理解
> LSTM通过三个"门"来控制信息的流入、保留和流出，解决了RNN的长程依赖问题。

#### 🍎 类比：带锁的记事本
```
LSTM有一个"长期记忆"（细胞状态 Cell State）和三个开关（门）：

遗忘门（Forget Gate）：决定"忘掉"哪些旧记忆
  → "这个信息以后用不到了，删掉"

输入门（Input Gate）：决定"记住"哪些新信息
  → "这个新信息很重要，加入记忆"

输出门（Output Gate）：决定"说出"哪些记忆
  → "现在需要用到记忆的哪个部分"
```

```
LSTM内部结构：

                   ┌──────────────────────────────────┐
                   │           细胞状态 Cₜ              │（长期记忆）
                   │  ×(遗忘)    +(输入)               │
xₜ, hₜ₋₁ →  [遗忘门f] [输入门i] [候选值g̃] [输出门o] → hₜ
```

### 3.5.3 GRU（门控循环单元）

> LSTM的简化版，只有两个门（重置门、更新门），参数更少，效果接近。
> 实践中：数据量小时GRU更好（参数少不易过拟合），数据量大时LSTM略优。

### 现实情况
> ⚠️ RNN/LSTM在NLP中已经被**Transformer**全面取代（见第4章）。但在时间序列预测、音频处理等领域仍有应用。面试中会考原理，但实际项目中用得越来越少。

#### ❓ 面试常问
**Q：LSTM如何解决RNN的梯度消失问题？**
> 💡 "LSTM引入了细胞状态（Cell State）作为长期记忆通道，遗忘门和输入门通过加法更新细胞状态（而非RNN的乘法叠加），使得梯度可以直接沿细胞状态通道传播，不会在长序列中指数级衰减。本质上，细胞状态提供了一条'梯度高速公路'，类似于ResNet的跳跃连接。"

## 3.6 常见问题与技巧 / Common Issues & Tricks

### 3.6.1 权重初始化

> 不能把权重初始化为全0——所有神经元会学到完全相同的东西（对称性问题）
> **Xavier初始化**：适合Sigmoid/Tanh
> **He初始化**：适合ReLU（更常用）

### 3.6.2 学习率调度 Learning Rate Scheduling

```
常见策略：
  固定学习率：简单但不是最优
  学习率衰减：随训练进行逐渐降低（StepLR、CosineAnnealingLR）
  Warmup：先从小学习率开始，逐渐增大，再衰减（Transformer训练必用）
  
  学习率 ↑
        │       ╭───╮
        │      ╱     ╲
        │     ╱       ╲___________
        │────╱  Warmup  衰减阶段
        └──────────────────────────→ 训练步数
```

### 3.6.3 早停 Early Stopping

```
监控验证集损失：

损失 ↑
     │                    ← 过拟合开始
     │  训练损失↘
     │       验证损失↘↗
     │              ↑
     │           在这里停止！
     └────────────────────→ Epoch

patience=10：验证损失连续10轮不下降就停止训练
```

### 3.6.4 超参数调优

| 超参数 | 典型范围 | 调法 |
|-------|---------|------|
| 学习率 | 1e-4 ~ 1e-1 | 最重要，先调这个 |
| Batch Size | 16 ~ 512 | 越大训练越稳，但需要更大显存 |
| 网络深度/宽度 | 视任务 | 由浅到深逐渐加 |
| Dropout率 | 0.1 ~ 0.5 | 过拟合严重时加大 |
| 权重衰减 | 1e-5 ~ 1e-2 | L2正则化强度 |

## 3.7 章节总结 / Chapter Summary

### 核心知识框架

```
深度学习 = 神经网络 + 反向传播 + 大数据 + 算力

训练流程：
  初始化参数 → 正向传播（预测）→ 计算损失 → 反向传播（求梯度）→ 优化器更新参数
  ↑_________________________________________________________________↓
                              循环N个Epoch

关键问题及解决方案：
  梯度消失  → ReLU + ResNet残差连接 + BatchNorm
  过拟合    → Dropout + L2正则化 + 早停 + 数据增强
  训练不稳定 → BatchNorm + 梯度裁剪 + 学习率调度
```

### 架构选型速查

| 数据类型 | 推荐架构 | 原因 |
|---------|---------|------|
| 图像 | CNN / ViT | 局部特征+空间不变性 |
| 文本/序列 | Transformer | 长程依赖+并行训练 |
| 时间序列 | LSTM / Transformer | 序列建模 |
| 通用 | MLP | 表格数据基线 |
